# VEYA — Analytics & KPIs

המדדים החשובים ביותר ל-VEYA. משמשים לתעדוף מוצר, אופטימיזציית המרה, וזיהוי
נטישה.

> **סטטוס אינסטרומנטציה `[לאימות]`:** לא ידוע איזה מהמדדים נמדד בפועל היום
> (יש/אין אנליטיקס מותקן?). שאלה פתוחה — `open-questions.md`. אל תניח שיש דאטה.

## מדד הצפון (North Star)
מועמד: **מספר חתונות שנוהלו בפועל דרך VEYA מקצה לקצה** (הזמנה → RSVP → הושבה).
זה תופס את הערך האמיתי, לא רק הרשמות. לאישור המייסד.

## Funnel — מדדי הפעלה והמרה
| מדד | מה הוא אומר |
|---|---|
| הרשמות | כמה נכנסו למערכת |
| אחוז השלמת Onboarding | כמה סיימו להקים אירוע מתוך שנרשמו |
| פתיחת אירועים | כמה אירועים אמיתיים נוצרו |
| **Activation** (ייבוא מוזמנים / הזמנה ראשונה) | הגיעו לרגע הערך הראשון |
| שליחת הזמנות | כמה אירועים שלחו בפועל |
| **אחוז אישורי הגעה** | מתוך ההזמנות שנשלחו — מדד ליבה מוצרי |
| שימוש בהושבה | כמה אירועים השתמשו במנוע/עורך המפה |
| שימוש במתנות באשראי (עתידי) | אימוץ עמוד המתנות |
| **המרות לתשלום** | כמה שילמו מתוך המשתמשים הפעילים |
| נטישה (Churn / Drop-off) | היכן ובאיזה שלב נוטשים |

## מדדי איכות/אמון
- זמן עד Activation (מהיר יותר = טוב יותר).
- שביעות רצון / NPS `[לאימות]`.
- אחוז מוזמנים שהשלימו RSVP בקליק (חוויית המוזמן).
- שיעור שגיאות/כשלים בשליחת הודעות.

## יעדים (Targets)
מדד בלי יעד הוא רק מספר. לכל מדד ליבה (אחוז אישורי הגעה, המרה לתשלום,
Activation) צריך יעד מספרי שמולו מודדים הצלחה.
> `[לאימות]` היעדים עצמם **טרם נקבעו** ודורשים החלטת מייסד + דאטה בסיס
> (baseline) אמיתי — `open-questions.md`. אל תמציא יעד; סמן כשאלה פתוחה.

## איך להשתמש בפועל
- כשמתכננים פיצ'ר: איזה מדד הוא אמור להזיז? אם אף אחד — שאל למה בונים.
- כשמאתרים נטישה: זהה את שלב ה-Funnel הבעייתי, הצלב עם `conversion-optimization.md`.
- לפני "נשפר את X" — ודא שיש דרך למדוד את X. אין? הוסף אינסטרומנטציה קודם.
